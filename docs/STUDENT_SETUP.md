# Student setup guide

This guide is for students and instructors who want to use the Maple Grove
GTA Synthea project without rebuilding Synthea from source.

Most users only need to:

1. clone the project with GitHub Desktop;
2. install Java 17;
3. download the prebuilt GTA Synthea JAR;
4. generate synthetic CSV files;
5. run the OpenEMR import process when that part is ready.

> All records generated by this project are synthetic. Do not represent them
> as real patient information.

---

## 1. Open the public repository

The repository is public, so no invitation is required.

1. Open this page in your browser:

<https://github.com/cabbagecanfly/maple-grove-synthea-openemr>

2. A GitHub account is optional for viewing or downloading the project.
3. A GitHub account is needed to clone through GitHub Desktop or contribute changes.

---

## 2. Install GitHub Desktop

Install GitHub Desktop for Windows or macOS, then sign in through the browser
using a GitHub account.

GitHub Desktop is used only to download the project and receive updates.
The Synthea and Python commands will still run in WSL or macOS Terminal.

---

## 3. Clone the project

In GitHub Desktop:

1. Select **File → Clone repository**.
2. Open the **GitHub.com** tab.
3. Select `maple-grove-synthea-openemr`.
4. Choose a local folder you can find easily.
5. Select **Clone**.

Do not place the repository inside another copy of the same project.

---

## 4. Install Java 17

Synthea requires a Java Development Kit. Use Java 17 for this project.

### Windows using WSL

Open WSL and run:

```bash
sudo apt update
sudo apt install -y openjdk-17-jdk-headless python3
```

Then verify:

```bash
java -version
python3 --version
```

Java should report version 17.

### macOS

Install a Java 17 JDK using a graphical `.pkg` installer.

Recommended settings:

- Java version: 17 LTS
- Package: JDK
- Apple Silicon Macs: aarch64
- Intel Macs: x64

After installation, open Terminal and verify:

```bash
java -version
python3 --version
```

---

## 5. Download the prebuilt GTA Synthea JAR

Most students should not rebuild Synthea from source.

1. Open the repository in a browser.
2. Open the **Releases** section.
3. Open the latest GTA Synthea Generator release.
4. Download:

```text
synthea-gta-maple-grove-v0.1.1.jar
```

5. Move the downloaded JAR directly into the project folder:

```text
tools/
```

The final location should look like:

```text
maple-grove-synthea-openemr/
└── tools/
    └── synthea-gta-maple-grove-v0.1.1.jar
```

Do not commit the JAR through GitHub Desktop.

---

## 6. Open the project in a terminal

### Windows using WSL

1. Open the cloned project folder in File Explorer.
2. Click the address bar.
3. Type:

```text
wsl
```

4. Press Enter.

### macOS

1. Open Terminal.
2. Type `cd` followed by one space.
3. Drag the cloned project folder from Finder into Terminal.
4. Press Enter.

Confirm that the terminal is inside the project:

```bash
pwd
```

---

## 7. Generate GTA synthetic patient CSV files

The final one-command generator will be added as:

```bash
python3 scripts/generate_gta_patients.py
```

Until that script is completed and tested, follow the current maintainer
instructions rather than inventing a different Java command.

Generated files will be written under:

```text
output/
```

---

## 8. Import selected data into OpenEMR

The OpenEMR importer is still under development.

The planned import order is:

1. patients;
2. encounters;
3. conditions;
4. allergies;
5. medications;
6. selected observations;
7. procedures and immunizations.

The importer will use the OpenEMR Standard REST API rather than writing
directly to the OpenEMR database.

---

## 9. Receive project updates

In GitHub Desktop:

1. Open this repository.
2. Select **Fetch origin**.
3. Select **Pull origin** when updates are available.

Before pulling, save or commit any local changes you want to keep.
