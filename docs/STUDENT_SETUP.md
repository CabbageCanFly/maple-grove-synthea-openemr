# Student setup guide

This guide is for students and instructors who want to use the Maple Grove
GTA Synthea project without rebuilding Synthea from source.

Most users only need to:

1. clone the project with GitHub Desktop;
2. install Java 17;
3. download the prebuilt GTA Synthea JAR;
4. generate synthetic CSV files;
5. run the currently implemented OpenEMR import scripts.

> All records generated by this project are synthetic. Do not represent them
> as real patient information.

---

## 1. Open the public repository

The repository is public, so no invitation is required.

1. Open this page in your browser:

<https://github.com/cabbagecanfly/maple-grove-synthea-openemr>

2. A GitHub account is optional for viewing or downloading the project.
3. A GitHub account is needed to clone through GitHub Desktop or contribute changes.

---

## 2. Install GitHub Desktop

Install GitHub Desktop for Windows or macOS, then sign in through the browser
using a GitHub account.

GitHub Desktop is used only to download the project and receive updates.
The Synthea and Python commands will still run in WSL or macOS Terminal.

---

## 3. Clone the project

In GitHub Desktop:

1. Select **File → Clone repository**.
2. Open the **GitHub.com** tab.
3. Select `maple-grove-synthea-openemr`.
4. Choose a local folder you can find easily.
5. Select **Clone**.

Do not place the repository inside another copy of the same project.

---

## 4. Install Java 17

Synthea requires a Java Development Kit. Use Java 17 for this project.

### Windows using WSL

Open WSL and run:

```bash
sudo apt update
sudo apt install -y openjdk-17-jdk-headless python3 python3-venv python3-venv
```

Then verify:

```bash
java -version
python3 --version
```

Java should report version 17.

### macOS

Install a Java 17 JDK using a graphical `.pkg` installer.

Recommended settings:

- Java version: 17 LTS
- Package: JDK
- Apple Silicon Macs: aarch64
- Intel Macs: x64

After installation, open Terminal and verify:

```bash
java -version
python3 --version
```

---

## 5. Download the prebuilt GTA Synthea JAR

Most students should not rebuild Synthea from source.

After opening the repository root in Terminal or WSL, run:

```bash
mkdir -p dist

curl -L \
  https://github.com/CabbageCanFly/maple-grove-synthea-openemr/releases/download/v0.1.1/synthea-gta-maple-grove-v0.1.1.jar \
  -o dist/synthea-gta-maple-grove-v0.1.1.jar
```

Verify:

```bash
ls -lh dist/synthea-gta-maple-grove-v0.1.1.jar
```

The final location is:

```text
dist/synthea-gta-maple-grove-v0.1.1.jar
```

Do not commit the JAR.

## 6. Open the project in a terminal

### Windows using WSL

1. Open the cloned project folder in File Explorer.
2. Click the address bar.
3. Type:

```text
wsl
```

4. Press Enter.

### macOS

1. Open Terminal.
2. Type `cd` followed by one space.
3. Drag the cloned project folder from Finder into Terminal.
4. Press Enter.

Confirm that the terminal is inside the project:

```bash
pwd
```

## 7. Prepare Python and generate GTA CSV files

Create and activate a project-local Python environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -r requirements_openemr_import.txt
```

Generate a fresh 100-patient dataset:

```bash
python3 scripts/generate_gta_patients.py --population 100
```

The generator:

- uses the pinned GTA Synthea JAR under `dist/`;
- uses `config/synthea-gta.properties`;
- creates a unique run below `output/runs/`;
- verifies the CSV files required by the supported importer;
- records hashes, row counts, and the selected CSV directory in
  `output/current-dataset.json`.

A different population size can be requested, for example:

```bash
python3 scripts/generate_gta_patients.py --population 25
```

Do not rename generated run folders merely to satisfy an importer default. The
orchestrator reads the current dataset manifest.

---

## 8. Configure OpenEMR and import the supported data

Detect the OpenEMR target:

```bash
python3 scripts/detect_openemr.py
```

For the local Docker target, ensure HTTPS is available:

```bash
python3 scripts/ensure_local_https.py
```

Register an OAuth client:

```bash
python3 scripts/register_openemr_client.py
```

Open OpenEMR and enable the newest client under:

```text
Administration -> System -> API Clients
```

Test the connection:

```bash
python3 scripts/test_openemr_connection.py
```

Preflight the selected dataset without creating records:

```bash
python3 scripts/import_openemr.py
```

Run the complete supported workflow:

```bash
python3 scripts/import_openemr.py \
  --commit \
  --quiet \
  --progress-every 100
```

The workflow imports patients, encounters, curated conditions, curated
allergies, medications, and supported vital signs in dependency order. Local
credentials and resumable maps are stored under `.local/` and must not be
committed.

Generic procedures, immunizations, care plans, devices, imaging studies, and
supplies are not redirected into unrelated OpenEMR resources when a suitable
writable API is unavailable.
